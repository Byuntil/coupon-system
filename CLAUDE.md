# Project Instructions

## Git Rules

- **Do NOT `git add` or `git commit` documentation files.** When creating or editing files under `docs/` or any `.md` documentation files, never stage or commit them unless the user explicitly requests it.

## Documentation Rules

- **After completing any implementation or technology/strategy selection, always add a "why" section to the relevant document under `docs/`.** This must include:
  - **Context**: What problem or requirement triggered this decision
  - **Alternatives considered**: What other options were evaluated and their trade-offs
  - **Decision**: What was ultimately chosen
  - **Rationale**: Why this choice was better than the alternatives
- Write all documentation content in **Korean**.
- Do not declare work complete without this documentation.
